<h2>{{ page.title }}</h2>

<p>
	There are sixteen general-purpose registers. The lower 32 bits, 16 bits, and 8 bits of each register are addressable.
	Also are eight 80-bit floating point and 64-bit MMX registers, of which are overlaid, sixteen 128bit XMM registers
	and one 64-bit <code>RFLAGS</code> register. The higher 32 bits of <code>RFLAGS</code> are reserved and unused.
</p>

<h3>Calling Conventions</h3>

<p> 
	With Windows x64 the calling convention is the Microsoft 64 calling convention used for C style function calling.
	Under Linux this would be called the Application Binary Interface (ABI).<br><br>
	The registers <code>rax</code>, <code>rcx</code>, <code>rdx</code>, <code>r8</code>, <code>r9</code>, 
	<code>r10</code>, <code>r11</code> are considered volatile (caller-saved).<br>
	The registers <code>rbx</code>, <code>rdi</code>, <code>rsi</code>, <code>rbp</code>, <code>rsp</code>, 
	<code>r12</code>, <code>r13</code>, <code>r14</code>, and <code>r15</code> are considered nonvolatile (callee-saved).
</p>

<h3>Microsoft 64 Calling Conventions</h3>

<ul>
	<li><code>rcx</code>, <code>rdx</code>, <code>r8</code>, <code>r9</code> are used for integer and pointer arguments 
		in that order left to right.
	</li>
	<li><code>xmm0</code>, <code>xmm1</code>, <code>xmm2</code>, and <code>xmm3</code> are used for 
		floating point arguments.
	</li>
	<li>Additional arguments are pushed on the stack left to right.</li>
	<li>It is the caller's responsibility to allocate 32 bytes of "shadow space" (for storing <code>rcx</code>, 
		<code>rdx</code>, <code>r8</code>, <code>r9</code> if needed) before calling the function.
	</li>
	<li>It is the caller's responsibility to clean the stack after the call.</li>
	<li>The stack is 16-byte aligned. The <code>call</code> instruction pushes an 8-byte return value, so the all non-leaf 
		functions must adjust the stack by a value of the form 16n+8 when allocating stack space.
	</li>
</ul>

<h3>Linux Application Binary Interface</h3>

<ul>
	<li>In user-level applications, the first six integer or pointer arguments are passed in registers 
		<code>rdi</code>, <code>rsi</code>, <code>rdx</code>, <code>rcx</code>, <code>r8</code>, <code>r9</code>.
	</li>
	<li>In kernel mode the interface is <code>rdi</code>, <code>rsi</code>, <code>rdx</code>, <code>r10</code>, 
		<code>r8</code>, <code>r9</code>.
	</li>
	<li>Integer return values up to 64 bits in size are stored in <code>rax</code> while values up to 
		128 bit are stored in <code>rax</code> and <code>rdx</code>.
	</li>
	<li>The number of the syscall has to be passed in register <code>rax</code>.</li>
	<li>A system-call is done via the <code>syscall</code> instruction. The kernel destroys registers 
		<code>rcx</code> and <code>r11</code>
	</li>
	<li>
		Returning from the <code>syscall</code>, register <code>rax</code> contains the result of the system-call.
		A value in the range between <code>-4095</code> and <code>-1</code> indicates an error, 
		it is <code>-errno</code>.
	</li>
</ul>
